# scopes.apis.io — OAuth Scopes Index

[**scopes.apis.io**](https://apis.io/scopes/) indexes the OAuth 2.0 scopes published by API providers across the [APIs.io](https://apis.io) network — the fine-grained permissions an application requests at authorization time to act against a provider's API.

**Live site:** [https://apis.io/scopes/](https://apis.io/scopes/)

## What's in this repo

- **`_scopes/`** — one Markdown record per provider (generated), holding that provider's OAuth schemes (flows + authorization/token URLs) and its full scope list (scope string, description, flows).
- **`index.html`, `_layouts/scope.html`, `_includes/`, `assets/`** — the browse (filter by provider or scope string) and detail pages
- **`_config.yml`** — Jekyll site configuration

## How it's built

Regenerated by [`network/scripts/build.py`](https://github.com/api-search/network/blob/main/scripts/build.py) on **every network build**: `process_provider` reads each provider's `scopes/<slug>-scopes.yml` artifact (type `OAuthScopes`, derived from the provider's OpenAPI security definitions and, where available, its documented scope reference) and writes one page per provider into `_scopes/`. The site is a Jekyll build deployed via the apis.io CloudFront origin.
